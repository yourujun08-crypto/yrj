I. Обзор проекта
Это система управления персоналом на Django 5.0+, поддерживающая управление отделами, управление сотрудниками, оформление отпусков и другие функции. В качестве базы данных используется SQLite (по умолчанию) или MySQL/PostgreSQL.

II. Структура проекта
Основная функциональность: personnel/ Этот каталог содержит реализацию основной функциональности проекта, в частности, включая:
migrations/: Файлы миграции базы данных (автоматически генерируются, используются для отслеживания изменений структуры базы данных);
models.py: Определения моделей (определяют модели данных, такие как отделы, сотрудники, запросы на отпуск и т. д.);
views.py: Функции представления (обрабатывают запросы пользователей и возвращают ответы);
urls.py: Конфигурация URL-адресов приложения (сопоставляет URL-адреса с соответствующими функциями представления);
templates/: Файлы шаблонов (HTML-страницы, используемые для отображения пользовательского интерфейса).

Каталог конфигурации проекта: hrm_system/ Этот каталог содержит глобальную конфигурацию проекта, в частности, включая:
settings.py: Настройки проекта (настраивает базу данных, статические файлы, промежуточное ПО и т. д.);
urls.py: Основная точка входа URL для проекта (определяет шаблон корневого URL и распределяет запросы к приложению);
wsgi.py: Конфигурация WSGI (используется для развертывания в производственной среде, например, Gunicorn/Apache). 
**Файл базы данных:** db.sqlite3 Файл базы данных SQLite по умолчанию, хранящий все данные проекта (например, отделы, сотрудники и записи об отпусках).

**Скрипт управления:** manage.py Скрипт управления Django, используемый для запуска сервера разработки (runserver), выполнения миграций базы данных (migrate), создания суперпользователей (createsuperuser) и т. д.

**Список зависимостей:** requirements.txt Содержит список необходимых зависимостей проекта (таких как Django, драйверы баз данных и т. д.), которые можно установить одним щелчком мыши с помощью команды `pip install -r requirements.txt`.
III. Требования к среде разработки

Python: 3.8+ (рекомендуется 3.11)

Django: 5.0+

База данных: SQLite (по умолчанию), MySQL 8.0+ или PostgreSQL 14+

Виртуальная среда: рекомендуется использовать venv (во избежание конфликтов зависимостей)
IV. Доступ к системе

http://127.0.0.1:8000/employees/

http://127.0.0.1:8000/departments/

http://127.0.0.1:8000/leaves/
V. Функциональное тестирование (проверка после эксплуатации)

Доступ к главной странице: Откройте браузер и введите http://127.0.0.1:8000/department. Должен отобразиться список отделов.

Добавление отдела: Нажмите кнопку «Добавить отдел» на главной странице, заполните название и описание, отправьте запрос и обновите страницу для просмотра.

Добавление сотрудника: Перейдите по адресу http://127.0.0.1:8000/employees/, нажмите кнопку «Добавить сотрудника» и заполните информацию о сотруднике (он должен быть связан с отделом).

Заявление на отпуск: Перейдите по адресу http://127.0.0.1:8000/leaves/, нажмите кнопку «Добавить отпуск» и заполните информацию о заявлении на отпуск (связанную с сотрудником).
